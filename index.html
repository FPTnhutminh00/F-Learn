Overview

WHAT IS CSS3?
CSS3 is the latest version of CSS Properties. The term CSS3 is not just a reference to new features in CSS, but the 3rd level in the progress of CSS Properties. CSS3 contains everything included in CSS2.1 (previous versions). It also adds new features to help developers solve some problems without non-semantic markup, complex scripts, or additional visuals.



What is the difference between CSS and CSS3?
The biggest change currently in CSS3 is the introduction of modules. The advantage of modules is that it allows the property to be completed and approved faster because segments are completed and approved in blocks.
Features included in CSS3 include support for additional selectors, drop shadows, rounded corners, multiple backgrounds, animations, transparency etc. It contains “CSS properties” (already included). subdivided into smaller parts). There are also new modules added. Some of the most important modules in CSS3 are:
- Selector
- Box model
- Backgrounds and borders
- Image value and alternative content
- Text effects
- 2D/3D Conversion
- Animations
- Multi-column layout
- User interface
- Most of the new CSS3 properties are implemented in new browsers.



What is HTML5?
HTML5 is recommended by the W3C from 2012 onwards. It includes processing models, Detailed Analysis Rules, Error Handling, Canvas for drawing in the browser, and Local Storage support.
HTML5 starts to support JavaScript API like Geolocation API to determine location, supporting cross-platform mobile apps.
HTML5 defines a single Markup language that can be written in HTML or XHTML language syntax and supports backward compatibility with previous HTML versions.



What is the difference between HTML and HTML5?
- HTML5 support for more applications: Some applications such as SVG, canvas ... are supported by HTML5, but used in HTML must use additional facilities.
- Store temporary data: HTML5 uses web SQL databases, application cache, and HTML only uses browser cache.
- JavaScript running in a web browser: HTML5 fully supports JavaScript running in a web browser, and HTML in older versions is not possible.
- SGML: Unlike HTML, HTML5 is not based on SGML, so the programming product is more compatible.
- Using MathML and SVG: HTML5 allows using MathML and SVG for text, but in HTML it is not supported.
- Elements: HTML5 integrates new and important elements like summary, time, aside, audio, command, data, datalist, details, embed, wbr, figcaption, figure, footer, header, article, hgroup, bdi, canvas, keygen, mark, meter, nav, output, progress, rp, rt, ruby, section, source, track, video... Besides, it also removes obsolete HTML elements like isindex, noframes, acronym, applet, basefont, dir, font, frame, frameset, big, center, strike….
